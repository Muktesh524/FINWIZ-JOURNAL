│                          DATABASE SCHEMA                                    │
└─────────────────────────────────────────────────────────────────────────────┘

news_articles Table:
├── id (INTEGER, PRIMARY KEY)
├── title (TEXT, NOT NULL)
├── description (TEXT)
├── link (TEXT, UNIQUE)
├── source (TEXT)
├── published_date (TIMESTAMP)
├── fetched_date (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)
├── approved (INTEGER, DEFAULT 0)
├── category (TEXT)
└── content_json (TEXT)

market_data Table:
├── id (INTEGER, PRIMARY KEY)
├── symbol (TEXT, NOT NULL)
├── price (REAL)
├── change_percent (REAL)
├── change_value (REAL)
├── fetched_date (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)
├── approved (INTEGER, DEFAULT 0)
└── data_json (TEXT)

generated_journals Table:
├── id (INTEGER, PRIMARY KEY)
├── created_date (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)
├── file_path (TEXT)
├── news_count (INTEGER)
├── market_data_count (INTEGER)
└── metadata_json (TEXT)

┌─────────────────────────────────────────────────────────────────────────────┐
│                          WORKFLOW SUMMARY                                   │
└─────────────────────────────────────────────────────────────────────────────┘

1. START APP
   run.bat (Windows) or run.sh (macOS/Linux)
                    ↓
2. FETCH DATA
   • Click "Fetch News" → Collects from 4 RSS feeds
   • Click "Fetch Market" → Gets prices from Yahoo Finance
   • Data stored in SQLite with status = 0 (pending)
                    ↓
3. REVIEW
   • Dashboard tab shows statistics
   • Fetch Data tab previews content
   • View counts, sources, categories
                    ↓
4. APPROVE
   • Individual approve/reject buttons
   • Bulk operations for efficiency
   • Status updated to 1 (approved)
                    ↓
5. GENERATE
   • Click "Generate Journal"
   • System queries approved content
   • Jinja2 renders HTML
   • File saved with timestamp
                    ↓
6. DOWNLOAD
   • Download button appears
   • Save to computer
   • Open in browser or share
                    ↓
7. OPTIONAL: CLEAR
   • Auto-clear approved content
   • Reset for next journal
   • Database remains intact
